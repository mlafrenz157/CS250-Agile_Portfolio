# CS250-Agile_Portfolio
CS 250 Portfolio Artifact

This repository contains my Sprint Review and Retrospective from the SNHU Travel Agile project. This project focused on applying Scrum practices, working through different roles, and adapting to changing requirements during development.

Reflection

When interpreting user needs, I found that user stories made a big difference. Instead of trying to build everything at once, user stories break things down into smaller pieces that are easier to understand and implement. It also helps keep the focus on what the user actually wants instead of what we assume they want. When user stories were unclear, it made everything harder, especially testing, so having clear requirements early matters more than I expected.

When developing programs, I tend to start with a general idea and then refine it as I go. After this course, I would want to use more Agile practices like breaking work into smaller tasks, getting feedback earlier, and adjusting as needed. Sprint-style thinking and continuous testing are probably the biggest things I would carry forward. It just makes it easier to handle changes without restarting everything.

Being a good team member in software development mostly comes down to communication. Sharing progress, asking questions when something is unclear, and making sure everyone is on the same page matters more than just writing code. One thing I noticed is that unclear communication slows everything down more than technical problems do. Working together and keeping things visible helps avoid that.
